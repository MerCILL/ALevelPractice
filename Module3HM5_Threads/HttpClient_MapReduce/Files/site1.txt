									<p>Learn how to use C# to create a MapReduce solution on HDInsight.</p>
<p>Apache Hadoop streaming allows you to run MapReduce jobs using a script or executable. Here, .NET is used to implement the mapper and reducer for a word count solution.</p>
<h2 id="net-on-hdinsight">.NET on HDInsight</h2>
<p>HDInsight clusters use <a href="https://mono-project.com" data-linktype="external">Mono (https://mono-project.com)</a> to run .NET applications. Mono version 4.2.1 is included with HDInsight version 3.6. For more information on the version of Mono included with HDInsight, see <a href="../hdinsight-component-versioning" data-linktype="relative-path">Apache Hadoop components available with HDInsight versions</a>.</p>
<p>For more information on Mono compatibility with .NET Framework versions, see <a href="https://www.mono-project.com/docs/about-mono/compatibility/" data-linktype="external">Mono compatibility</a>.</p>
<h2 id="how-hadoop-streaming-works">How Hadoop streaming works</h2>
<p>The basic process used for streaming in this document is as follows:</p>
<ol>
<li>Hadoop passes data to the mapper (<em>mapper.exe</em> in this example) on STDIN.</li>
<li>The mapper processes the data, and emits tab-delimited key/value pairs to STDOUT.</li>
<li>The output is read by Hadoop, and then passed to the reducer (<em>reducer.exe</em> in this example) on STDIN.</li>
<li>The reducer reads the tab-delimited key/value pairs, processes the data, and then emits the result as tab-delimited key/value pairs on STDOUT.</li>
<li>The output is read by Hadoop and written to the output directory.</li>
</ol>
<p>For more information on streaming, see <a href="https://hadoop.apache.org/docs/r2.7.1/hadoop-streaming/HadoopStreaming.html" data-linktype="external">Hadoop Streaming</a>.</p>
<h2 id="prerequisites">Prerequisites</h2>
<ul>
<li><p>Visual Studio.</p>
</li>
<li><p>A familiarity with writing and building C# code that targets .NET Framework 4.5.</p>
</li>
<li><p>A way to upload .exe files to the cluster. The steps in this document use the Data Lake Tools for Visual Studio to upload the files to primary storage for the cluster.</p>
</li>
<li><p>If using PowerShell, you'll need the <a href="/en-us/powershell/azure/" data-linktype="absolute-path">Az Module</a>.</p>
</li>
<li><p>An Apache Hadoop cluster on HDInsight. See <a href="apache-hadoop-linux-tutorial-get-started" data-linktype="relative-path">Get Started with HDInsight on Linux</a>.</p>
</li>
<li><p>The URI scheme for your clusters primary storage. This scheme would be <code>wasb://</code> for Azure Storage, <code>abfs://</code> for Azure Data Lake Storage Gen2 or <code>adl://</code> for Azure Data Lake Storage Gen1. If secure transfer is enabled for Azure Storage or Data Lake Storage Gen2, the URI would be <code>wasbs://</code> or <code>abfss://</code>, respectively.</p>
</li>
</ul>
<h2 id="create-the-mapper">Create the mapper</h2>
<p>In Visual Studio, create a new .NET Framework console application named <em>mapper</em>. Use the following code for the application:</p>
<pre><code class="lang-csharp">using System;
using System.Text.RegularExpressions;

namespace mapper
{
    class Program
    {
        static void Main(string[] args)
        {
            string line;
            //Hadoop passes data to the mapper on STDIN
            while((line = Console.ReadLine()) != null)
            {
                // We only want words, so strip out punctuation, numbers, etc.
                var onlyText = Regex.Replace(line, @&quot;\.|;|:|,|[0-9]|'&quot;, &quot;&quot;);
                // Split at whitespace.
                var words = Regex.Matches(onlyText, @&quot;[\w]+&quot;);
                // Loop over the words
                foreach(var word in words)
                {
                    //Emit tab-delimited key/value pairs.
                    //In this case, a word and a count of 1.
                    Console.WriteLine(&quot;{0}\t1&quot;,word);
                }
            }
        }
    }
}
</code></pre>
<p>After you create the application, build it to produce the <em>/bin/Debug/mapper.exe</em> file in the project directory.</p>
<h2 id="create-the-reducer">Create the reducer</h2>
<p>In Visual Studio, create a new .NET Framework console application named <em>reducer</em>. Use the following code for the application:</p>
<pre><code class="lang-csharp">using System;
using System.Collections.Generic;

namespace reducer
{
    class Program
    {
        static void Main(string[] args)
        {
            //Dictionary for holding a count of words
            Dictionary&lt;string, int&gt; words = new Dictionary&lt;string, int&gt;();

            string line;
            //Read from STDIN
            while ((line = Console.ReadLine()) != null)
            {
                // Data from Hadoop is tab-delimited key/value pairs
                var sArr = line.Split('\t');
                // Get the word
                string word = sArr[0];
                // Get the count
                int count = Convert.ToInt32(sArr[1]);

                //Do we already have a count for the word?
                if(words.ContainsKey(word))
                {
                    //If so, increment the count
                    words[word] += count;
                } else
                {
                    //Add the key to the collection
                    words.Add(word, count);
                }
            }
            //Finally, emit each word and count
            foreach (var word in words)
            {
                //Emit tab-delimited key/value pairs.
                //In this case, a word and a count of 1.
                Console.WriteLine(&quot;{0}\t{1}&quot;, word.Key, word.Value);
            }
        }
    }
}
</code></pre>
<p>After you create the application, build it to produce the <em>/bin/Debug/reducer.exe</em> file in the project directory.</p>
<h2 id="upload-to-storage">Upload to storage</h2>
<p>Next, you need to upload the <em>mapper</em> and <em>reducer</em> applications to HDInsight storage.</p>
<ol>
<li><p>In Visual Studio, select <strong>View</strong> &gt; <strong>Server Explorer</strong>.</p>
</li>
<li><p>Right-click <strong>Azure</strong>, select <strong>Connect to Microsoft Azure Subscription...</strong>, and complete the sign-in process.</p>
</li>
<li><p>Expand the HDInsight cluster that you wish to deploy this application to. An entry with the text <strong>(Default Storage Account)</strong> is listed.</p>
<p><span class="mx-imgBorder">
<img src="media/apache-hadoop-dotnet-csharp-mapreduce-streaming/hdinsight-storage-account.png" alt="Storage account, HDInsight cluster, Server Explorer, Visual Studio" data-linktype="relative-path">
</span>
</p>
<ul>
<li><p>If the <strong>(Default Storage Account)</strong> entry can be expanded, you're using an <strong>Azure Storage Account</strong> as default storage for the cluster. To view the files on the default storage for the cluster, expand the entry and then double-click <strong>(Default Container)</strong>.</p>
</li>
<li><p>If the <strong>(Default Storage Account)</strong> entry can't be expanded, you're using <strong>Azure Data Lake Storage</strong> as the default storage for the cluster. To view the files on the default storage for the cluster, double-click the <strong>(Default Storage Account)</strong> entry.</p>
</li>
</ul>
</li>
<li><p>To upload the .exe files, use one of the following methods:</p>
<ul>
<li><p>If you're using an <strong>Azure Storage Account</strong>, select the <strong>Upload Blob</strong> icon.</p>
<p><span class="mx-imgBorder">
<img src="media/apache-hadoop-dotnet-csharp-mapreduce-streaming/hdinsight-upload-icon.png" alt="HDInsight upload icon for mapper, Visual Studio" data-linktype="relative-path">
</span>
</p>
<p>In the <strong>Upload New File</strong> dialog box, under <strong>File name</strong>, select <strong>Browse</strong>. In the <strong>Upload Blob</strong> dialog box, go to the <em>bin\debug</em> folder for the <em>mapper</em> project, and then choose the <em>mapper.exe</em> file. Finally, select <strong>Open</strong> and then <strong>OK</strong> to complete the upload.</p>
</li>
<li><p>For <strong>Azure Data Lake Storage</strong>, right-click an empty area in the file listing, and then select <strong>Upload</strong>. Finally, select the <em>mapper.exe</em> file and then select <strong>Open</strong>.</p>
</li>
</ul>
<p>Once the <em>mapper.exe</em> upload has finished, repeat the upload process for the <em>reducer.exe</em> file.</p>
</li>
</ol>
<h2 id="run-a-job-using-an-ssh-session">Run a job: Using an SSH session</h2>
<p>The following procedure describes how to run a MapReduce job using an SSH session:</p>
<ol>
<li><p>Use <a href="../hdinsight-hadoop-linux-use-ssh-unix" data-linktype="relative-path">ssh command</a> to connect to your cluster. Edit the command below by replacing CLUSTERNAME with the name of your cluster, and then enter the command:</p>
<pre><code class="lang-cmd">ssh sshuser@CLUSTERNAME-ssh.azurehdinsight.net
</code></pre>
</li>
<li><p>Use one of the following commands to start the MapReduce job:</p>
<ul>
<li><p>If the default storage is <strong>Azure Storage</strong>:</p>
<pre><code class="lang-bash">yarn jar /usr/hdp/current/hadoop-mapreduce-client/hadoop-streaming.jar \
    -files wasbs:///mapper.exe,wasbs:///reducer.exe \
    -mapper mapper.exe \
    -reducer reducer.exe \
    -input /example/data/gutenberg/davinci.txt \
    -output /example/wordcountout
</code></pre>
</li>
<li><p>If the default storage is <strong>Data Lake Storage Gen1</strong>:</p>
<pre><code class="lang-bash">yarn jar /usr/hdp/current/hadoop-mapreduce-client/hadoop-streaming.jar \
    -files adl:///mapper.exe,adl:///reducer.exe \
    -mapper mapper.exe \
    -reducer reducer.exe \
    -input /example/data/gutenberg/davinci.txt \
    -output /example/wordcountout
</code></pre>
</li>
<li><p>If the default storage is <strong>Data Lake Storage Gen2</strong>:</p>
<pre><code class="lang-bash">yarn jar /usr/hdp/current/hadoop-mapreduce-client/hadoop-streaming.jar \
    -files abfs:///mapper.exe,abfs:///reducer.exe \
    -mapper mapper.exe \
    -reducer reducer.exe \
    -input /example/data/gutenberg/davinci.txt \
    -output /example/wordcountout
</code></pre>
</li>
</ul>
<p>The following list describes what each parameter and option represents:</p>
<table>
<thead>
<tr>
<th>Parameter</th>
<th>Description</th>
</tr>
</thead>
<tbody>
<tr>
<td>hadoop-streaming.jar</td>
<td>Specifies the jar file that contains the streaming MapReduce functionality.</td>
</tr>
<tr>
<td>-files</td>
<td>Specifies the <em>mapper.exe</em> and <em>reducer.exe</em> files for this job. The <code>wasbs:///</code>, <code>adl:///</code>, or <code>abfs:///</code> protocol declaration before each file is the path to the root of default storage for the cluster.</td>
</tr>
<tr>
<td>-mapper</td>
<td>Specifies the file that implements the mapper.</td>
</tr>
<tr>
<td>-reducer</td>
<td>Specifies the file that implements the reducer.</td>
</tr>
<tr>
<td>-input</td>
<td>Specifies the input data.</td>
</tr>
<tr>
<td>-output</td>
<td>Specifies the output directory.</td>
</tr>
</tbody>
</table>
</li>
<li><p>Once the MapReduce job completes, use the following command to view the results:</p>
<pre><code class="lang-bash">hdfs dfs -text /example/wordcountout/part-00000
</code></pre>
<p>The following text is an example of the data returned by this command:</p>
<pre><code class="lang-output">you     1128
young   38
younger 1
youngest        1
your    338
yours   4
yourself        34
yourselves      3
youth   17
</code></pre>
</li>
</ol>
<h2 id="run-a-job-using-powershell">Run a job: Using PowerShell</h2>
<p>Use the following PowerShell script to run a MapReduce job and download the results.</p>
<pre><code class="lang-powershell" name="main"># Login to your Azure subscription
$context = Get-AzContext
if ($context -eq $null) 
{
    Connect-AzAccount
}
$context

# Get HDInsight info
$clusterName = Read-Host -Prompt &quot;Enter the HDInsight cluster name&quot;
$creds=Get-Credential -Message &quot;Enter the login for the cluster&quot;

# Path for job output
$outputPath=&quot;/example/wordcountoutput&quot;

# Progress indicator
$activity=&quot;C# MapReduce example&quot;
Write-Progress -Activity $activity -Status &quot;Getting cluster information...&quot;
#Get HDInsight info so we can get the resource group, storage, etc.
$clusterInfo = Get-AzHDInsightCluster -ClusterName $clusterName
$resourceGroup = $clusterInfo.ResourceGroup
$storageActArr=$clusterInfo.DefaultStorageAccount.split('.')
$storageAccountName=$storageActArr[0]
$storageType=$storageActArr[1]

# Progress indicator
#Define the MapReduce job
# Note: using &quot;/mapper.exe&quot; and &quot;/reducer.exe&quot; looks in the root
#       of default storage.
$jobDef=New-AzHDInsightStreamingMapReduceJobDefinition `
    -Files &quot;/mapper.exe&quot;,&quot;/reducer.exe&quot; `
    -Mapper &quot;mapper.exe&quot; `
    -Reducer &quot;reducer.exe&quot; `
    -InputPath &quot;/example/data/gutenberg/davinci.txt&quot; `
    -OutputPath $outputPath

# Start the job
Write-Progress -Activity $activity -Status &quot;Starting MapReduce job...&quot;
$job=Start-AzHDInsightJob `
    -ClusterName $clusterName `
    -JobDefinition $jobDef `
    -HttpCredential $creds

#Wait for the job to complete
Write-Progress -Activity $activity -Status &quot;Waiting for the job to complete...&quot;
Wait-AzHDInsightJob `
    -ClusterName $clusterName `
    -JobId $job.JobId `
    -HttpCredential $creds

Write-Progress -Activity $activity -Completed

# Download the output 
if($storageType -eq 'azuredatalakestore') {
    # Azure Data Lake Store
    # Fie path is the root of the HDInsight storage + $outputPath
    $filePath=$clusterInfo.DefaultStorageRootPath + $outputPath + &quot;/part-00000&quot;
    Export-AzDataLakeStoreItem `
        -Account $storageAccountName `
        -Path $filePath `
        -Destination output.txt
} else {
    # Az.Storage account
    # Get the container
    $container=$clusterInfo.DefaultStorageContainer
    #NOTE: This assumes that the storage account is in the same resource
    #      group as HDInsight. If it is not, change the
    #      --ResourceGroupName parameter to the group that contains storage.
    $storageAccountKey=(Get-AzStorageAccountKey `
        -Name $storageAccountName `
    -ResourceGroupName $resourceGroup)[0].Value

    #Create a storage context
    $context = New-AzStorageContext `
        -StorageAccountName $storageAccountName `
        -StorageAccountKey $storageAccountKey
    # Download the file
    Get-AzStorageBlobContent `
        -Blob 'example/wordcountoutput/part-00000' `
        -Container $container `
        -Destination output.txt `
        -Context $context
}
</code></pre>
<p>This script prompts you for the cluster login account name and password, along with the HDInsight cluster name. Once the job completes, the output is downloaded to a file named <em>output.txt</em>. The following text is an example of the data in the <code>output.txt</code> file:</p>
<pre><code class="lang-output">you     1128
young   38
younger 1
youngest        1
your    338
yours   4
yourself        34
yourselves      3
youth   17
</code></pre>
